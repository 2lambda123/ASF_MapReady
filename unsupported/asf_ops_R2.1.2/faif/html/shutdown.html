How to shut down FAIF

1. Save and then stop crontab
2. Check for active jobs
3. Stop FAxmitserver

1. Save and then stop crontab

To save a copy of the active crontab file, use a command like: 

     >crontab -l > crontab_file_save

After saving the crontab file, stop the crontab with this command: 

     >crontab -r

When you're ready to re-instate this crontab file, use this command: 

     >crontab crontab_file_save

2. Check for active jobs

Find out what's running: 

       >ps -ef | grep faif

If at all possible, FAIF shell scripts which are executing should be allowed to finish. Interrupting one may require careful manual attention before re-starting. Any script lock files will also have to be manually removed when you are ready to resume.

Interrupting FAIF directory monitors should also be avoided. However, they should be able to recover cleanly if you must stop them.

3. Stop FAxmitserver

Kill the server: 

 
        >ps -ef | grep FAxmitserver
        >kill -2 #

When you're ready to restart the server: 

  >$FAIF_BINPATH/FAxmitserver -c $FAIF_ROOTPATH/config/FAxmitserver.config \
    -e $FAXMIT_CDS_ENTRY &
