A few hints and reminders about crontab

Summary :

For full details on cron and crontab, refer to their man pages.

1. Listing and reading a crontab file
2. Editing and replacing a crontab file
3. Managing alternate crontab files

1. Listing and reading a crontab file

Use the command 

    >crontab -l
to list the active crontab file.

Each line consists of a program execution schedule followed by the string which will be executed by the shell at the specified time(s). Lines that begin with # are comment lines and are ignored by the cron daemon.

The first five items on the line are: 

    minute (0-59) 
    hour (0-23) 
    day of the month (1-31) 
    month of the year (1-12) 
    day of the week (0-6 with 0=Sunday)
Any of these may be an asterisk, meaning all legal values. These five items must be separated by spaces or tabs. The items may also be ranges (such as 2-5) or sequences (such as 0,15,30,45)

For example, this line: 

      16 03,09,15,21 * * * /asf/bin/CSAgetfile_all.csh
will run the shell script CSAgetfile_all.csh at 3:16am, 9:16am, 3:16pm, and 9:16pm every day.

2. Editing and replacing a crontab file

Use the command 

     >crontab -l > crontab.sav
to save a copy of the current active crontab file in crontab.sav.

Use the command 

     >crontab new_crontab_filename
to make new_crontab_filename become the active crontab file.

Use the command 

     >setenv EDITOR my_favorite_editor
to establish the editor you wish to use when working on the active crontab file (should you choose to do so).

Use the command 

     >crontab -e
to edit the active crontab file, using the editor specified in the EDITOR environment variable.

3. Managing alternate crontab files

You may wish to make certain changes, perhaps repeatedly, to the scheduled FAIF program execution. Perhaps one external agency is only sporadically available. Or the communication link to another will be down for a week. Or due to personnel vacation schedules, certain FAIF jobs must be deferred for a time. Or some FAIF software is about to be upgraded.

As an alternative to repeatedly editing the active crontab file, you may decide to keep copies of several alternate crontab files ready for use. These files may have been created from scratch, or from editing the default crontab file, /local/faif/crontab.default, or from editing and then saving a copy of the active crontab file.

To save a copy of the active crontab file, use a command like: 

     >crontab -l > crontab_file_full_version

To make another file become the active crontab file, use a command like: 

     >crontab crontab_file_no_CSA
