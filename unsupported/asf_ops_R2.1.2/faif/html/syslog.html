Reading FAIF syslog entries

syslog is a unix utility. FAIF logs all of its activities in syslog.

1. Locating and reading the FAIF syslog
2. syslog messages for dirmon activity
3. syslog messages for CSAgetfile activity
4. syslog messages for FAxmit activity

1. Locating and reading the FAIF syslog

The FAIF operator's .cshrc file should contain the following line: 

    >source syslog_aliases.csh
This establishes the alias $SYS_LOG_NAME for the syslog filename and several aliases to help you browse the log. To list only the latest segment of the syslog, use:
    >tail_log
To load a copy of the FAIF syslog into an emacs editor window, use: 
    >ed_log
To use some other editor or browser with the FAIF syslog, use the $SYS_LOG_NAME alias.

Each line begins with the date and time, the machine name, the program name, and the process ID. This is usually followed by the severity indication (NOTICE, WARNING, etc.) and the text of the log message.

Errors and warnings in syslog are the best starting point when trouble-shooting FAIF.

2. syslog messages for dirmon activity

In the nominal case, the log messages from FAIF dirmon programs will follow this outline to handle one file: 


  Establishing a semaphore.
  "Routing file filename."
  Processing and possible translation, according to filetype.
  IMS archive ("added" or "replaced") for the file and any translation files.
  Adding file to file list log.
  "Routed filename."
  "No files in file list."

When the directory being monitored has more than 1 file, the syslog lines from "Routing file filename" to "Routed filename" will be repeated for each file. Finally, the message "No files in file list" indicates that every file in the directory has been processed.

3. syslog messages for CSAgetfile activity

In the nominal case, the log messages from CSAgetfile will follow this outline: 


  Establishing a semaphore
  Normal child process termination, exit status = 0
  Normal termination obtaining or checking for file type from remote host.
  Normal child process termination, exit status = 0
  Routing file filename.
  Processing and possible translation, according to filetype.
  IMS archive ("replacement") for the file and any translation files.
  Adding file to file list log.
  Routed filename.
  Destroyed semaphore...

When there is more than 1 file of the specified file type, the syslog lines from "Routing file filename" to "Routed filename" will be repeated for each file.

4. syslog messages for FAxmit activity

In the nominal case, the log messages from FAxmit will look like this: 


  Server entry ... is listening for remote procedure calls.
  Normal child process termination, exit status = 0.
  Successfully transferred filename to destination.
